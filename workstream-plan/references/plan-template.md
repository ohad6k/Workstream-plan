# Workstream Plan Template

Use this template for large product workstream plans.

## 0. Output Artifact

Write the full plan to a Markdown file.

At the top include:

- title
- date
- objective
- status
- source prompt summary

## 1. Product Direction

- Vision
- Problem statement
- Target user
- Desired outcome
- Success definition

## 2. Scope

- In scope
- Out of scope
- Constraints
- Assumptions
- Decision boundaries

## 3. User Journey

- Entry points
- Main flows
- Friction points
- Failure cases
- User confidence gaps
- Adoption concerns

## 4. Workstream Map

List all major workstreams with:

- name
- purpose
- owner suggestion
- relative priority
- dependencies
- milestone target

## 5. Full Workstreams

For each workstream include:

- objective
- strategic value
- detailed task breakdown
- phases
- dependencies
- blockers
- risks
- fallback path
- acceptance signals
- open questions

## 6. Sequencing

- Phase 1
- Phase 2
- Phase 3
- Launch path
- Post-launch path

## 7. Risks

- product risks
- user risks
- adoption risks
- execution risks
- positioning risks

## 8. Rollout

- internal rollout
- external rollout
- communication plan
- feedback loop
- adjustment path

## 9. Open Questions

- unresolved decisions
- decisions to make early
- decisions to defer

## 10. Exclusions

By default do not include:

- API contracts
- schema definitions
- repo-specific implementation tasks
- code-level architecture plans
- production deployment detail
